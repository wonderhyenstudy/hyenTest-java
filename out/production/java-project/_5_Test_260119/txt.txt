상속 중요함 *************************************

업캐스팅     : 위로 던진다. 작은타입 -> 큰타입. 자식 -> 부모. 부모타입으로 형 변환
다운캐스팅   : 큰다입 -> 작은타입. 부모 -> 자식. 자식타입으로 형 변환
instanceof : 객체타입 확인방법. 객체타입 안전하게 확인하는 방법
오버라이딩   : 부모 메서드를 자식 클래스에서 재정의하고 다형성을 구현할수 있다.




추상 클래스 : 1번만 상속받을 수 있다.
            정의방법 : 클래스 앞에 Abstract
인터페이스  : 약속. 여러개 구현 가능. 강제성.
            정의방법 : Interface 사용
그렇다면, 언제 추상클래스를 쓰고, 언제 인터페이스를 써야 하는가?
'정체성'을 정의하려면 상속/추상클래스를,
'능력/규격'을 정의하려면 인터페이스를 선택

| 구분        | 추상 클래스 | 인터페이스 |
| ------ | ---------------- | -------------------------------------------- |
| 키워드      | `abstract class` | `interface` |
| 상속/구현   | `extends` | `implements` |
| 메서드 구현 | 일부 구현 가능 | 자바 8 이전까지는 전부 추상, 이후는 `default`, `static` 가능 |
| 다중 구현   | 불가 (1개만 상속) | 다중 구현 가능 |
| 용도       | 공통 기능 + 일부 구현 | 기능 규약 정의, 설계 기반 제공 |
