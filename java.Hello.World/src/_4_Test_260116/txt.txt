
명칭과 뜻을 정확히 구분하자!!
=============================================================
클래스     : 객체를 만들기 위한 설계도
객체       : 클래스로부터 만들어진 실체. 인스턴스
생성자     : 객체 생성시 자동으로 호출되는 특별한 메서드
필드(변수), 메서드(기능)     : 객체가 갖는 상태와 동작
클래스명 = 생성자명          : 같아야 한다
기본생성자/메인생성자/서브생성자   : 하나일경우 기본생성자 없어도 된다.
메서드 = 함수     : 만들어 여기저기 사용. 사용법 : 함수명() or 함수명(인자값)
매개변수    : 받는변수(나 이런 종류의 데이터 필요해. 미리 정해둔 변수명. 데이터 형태가 정해진 그릇)
반환값 = 리턴값   : 메서드 작업후 결과물
인수 = 인자값    : 메서드 사용할때 매개변수에 전달되는 실제값(구체적인 데이터값)
객체 생성   : 만든 클래스를 사용하는 행위
           : [클래스명] [새로운이름] = new [클래스명]();
             [클래스명] [새로운이름] = new [클래스명](인수);
배열      : 동일한 타입의 데이터들을 하나의 변수에 담아 순서대로 관리하는 자료구조(아파트 단지)



기본값/사용방법
=============================================================
변수  : 데이터를 저정하는 공간
    int 변수명;        // 1. 변수선언(정수를 담을 변수명 이라는 공간을 만든다)
    변수명 = 30;       // 2. 변수명에 초기값 30을 대입한다. 초기화
    System.out.println(변수명); // 3. 30이라는 변수명에 대입된 값을 출력한다

    정수         : int (기본), long (매우 큰 숫자)
    실수 (소수점) : double
    문자/문자열   : char (문자 하나), String (문장)
    논리 (참/거짓): boolean (true 또는 false)

    지역 변수    : 메서드 내부에서 선언. 메서드 끝나면 사라짐
    인스턴스 변수 : 클래스 안, 메서드 밖에서 선언하며 객체가 살아 있는 동안 유지
    클래스 변수   : static 키워드가 붙으면, 프로그램 시작부터 끝까지 공유

배열
    int[] 변수명 = new int[5]              // int 형식 변수명 배열에 int 형식의 데이터 5개의 공간을 만든다
    int[] 변수명 = {10, 20, 30, 40, 50}    // int 형식 변수명 배열에 int 형식의 데이터 각각의 값(10, 20, 30, 40, 50)을 각각의 공간에 넣어 만든다
    변수명[0] = 95;                        // 첫 번째 칸(0번 인덱스)의 값을 95로 변경
    System.out.println(변수명[1]);        // 두 번째 칸의 값(85) 출력

배열과 함께 사용하는 for문
    String[] names = {"Alice", "Bob", "Charlie"};
    // 향상된 for문: "names 배열의 값을 하나씩 name에 담아라"
    for (String name : names) {
        System.out.println("이름: " + name);
    }

함께 사용되는것들
    배열명.length      : 배열 안의 갯수를 구할때 사용


변수배열
    int[] 변수명 = new in[3]

객체배열
    클래스명[] 새변수명 = new 클래스명[]
    클래스명[] 새변수명 = new 클래스명[인수]


접근지정자
    public      : 완전 공개. 어디서든 접근 가능
    private     : 비공개. 직접접근이 안된다. set or get 클래스로 필요
    static      : 공용. 프로그램 안에서 전부 가능
    final       : 상수. 변하지 않는 값

상속         : 코드 재사용 목적 / 단일상속이 기본, 다중 상속은 인터페이스로 대체
extends     : 부모에서 자식으로 상속
              자식클래스명 extends 부모 클래스명
super       : 부모 생성, 메서드에 접근할때 사용
@Override   : 메서드를 재정의 할때 명시하고 한다
protected   : 자식 클래스에서 접근 가능
               protected String 변수명;
